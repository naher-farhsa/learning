IOC : Inversion of control is about controlling the creation, lifecycle and wiring of object by external source. (Idea) 

DI : Dependency Injection is about using object by getting it from external source rather than getting by itself. (Usage)

Spring : A Java framework that implement ioc via di (Implementation)

Bean Life-Cycle:
- Bean Instantiation
- Prop Injection
- Set Bean Name
- Set Bean Factory
- Set App Context
- Pre-Initialize (@PreConstruct)
- Initialize 
- Post-Initialize (@PostConstruct)
- Pre-Destroy (@Pre-Destroy)
- Destroy


Mode of DI: 
- Manual 
- Container( Spring-IOC )

Types of DI:
- Constructor Injection (recommended -> avoid null, immutablity, detect cyclic dependency)
- Setter Injection -> (mutable)
- Field Injection -> (code readability)

IOC Container :
- Bean Factory
- Application Context  getBean()

Bean: A object of any class/component whose is managed by Spring IOC container.

NOTE: Spring can only inject beans into beans

Bean meta data config :                              
- XML based        -> External, declarative wiring (Manual), <Bean>
- Java Based       -> Explicit, type safe wiring   (Manual), @Bean + @Configuration
- Annotation based -> Implicit, convention wiring   (Auto-wiring), @Component + @Service + @Repository + @ComponentScan + @Configuration

ClassPathXmlApplicationContext, AnnotationConfigApplicationContext


Bean Scope:
- Singleton -> Single bean per use for a class
- Prototype -> New bean per use for a class

Bean Resolution:
- By Type -> Checks for bean class/interface type (Use if one bean of a type exist)
- By Name -> Checks for bean name/id (Use if multiple bean of a type exist)

- Primary -> Mark a bean default choice during bean resolution
- Qualifier -> Mark a bean first choice during bean resolution 

NOTE : @Qualifier > @Primary

- @Resource -> inject bean by name
- @Autowired -> inject beans by type
- @PropertySource -> load property files
- @Value -> values to field from property files

Spring Configuration :
- application.property -> central config file for entire spring boot project
